## 1. Детектирование ключевых точек
Сначала было загружено 8 изображения для дальнейшей работы.

Далее применяется гауссово размытие с ядром 5×5 и sigma=3 для того чтобы уменьшить влияние шума на вычисление градиентов.

После для получения производных по X и Y была использована свертка Собеля. Оператор Собеля - дифференциальный оператор, который использует 2 ядра фильтров для фильтрации изображений вдоль 2-х направлений. Он делает это, вычисляя изменения яркости (градиента) между соседними пикселями, что позволяет определить, где находятся границы объектов. Так находится направление наибольшего увеличения яркости и величина её изменения в этом направлении.

Градиент показывает, насколько быстро изменяется яркость изображения в определённой точке. Если градиент велик, это указывает на резкое изменение яркости, что часто соответствует границе объекта. 

Свёртка - это математическая операция, которая применяется к изображению с использованием специального фильтра (ядра). В случае оператора Собеля используются два разных ядра для выявления изменений в горизонтальном и вертикальном направлениях.

Изображение свёртывается с каждым из ядер. Это означает, что каждое ядро проходит по изображению и вычисляет новое значение для каждого пикселя, основываясь на значениях соседних пикселей.

Градиент по X - выделяет вертикальные края (горизонтальные изменения яркости). Светлые области => изменение яркости слева направо, темные => справа налево.

Градиент по Y - выделяет горизонтальные края (вертикальные изменения яркости). Светлые области => изменение яркости сверху вниз, темные => снизу вверх.

Магнитуда градиента - показывает силу края (насколько резкий переход). Чем ярче пиксель, тем сильнее край.

Ориентация градиента - показывает направление края. Разные цвета соответствуют разным углам (0° = красный, 90° = зеленый, 180° = голубой, 270° = пурпурный).

Построение структурного тензора. Далее для каждого пикселя вычисляются три компонента: dx², dy², dx·dy.

Структурный тезор (известен также как матрица вторых моментов) - это матрица, полученная из градиента функции. Ее вид:

Далее переходим к вычислению собственных значений. Для каждой матрицы находятся собственные значения λ₁ и λ₂. Записываем в пиксель минимальное собственное значение матрицы М. Почему:

Оба значения маленькие => плоская область

Одно большое, другое маленькое => край 

Оба большие => угол (значительные изменения в двух направлениях)

Следовательно, на углах оба собственных значения большие, на краях - одно большое, другое маленькое. Минимальное значение - хорошая мера "угловатости" (чем больше, тем более похоже на угол). Такой подход гарантирует, что углом считается только точка с сильными изменениями во всех направлениях.

# Вычисление дескрипторов (SIFT-подобный метод)
SIFT (Scale-Invariant Feature Transform) - это алгоритм для выделения и описания ключевых точек, инвариантный к масштабу и повороту.

Дескриптор - идентификатор ключевой точки, выделяющий её из остальной массы особых точек, описывает ее уникальные характеристики. 

Сначала происходит извлечение патча. Вокруг каждой ключевой точки вырезается область 16×16 пикселей. Проверяется, чтобы патч полностью помещался в изображении (точка не слишком близко к краю).

Вычисление градиентов внутри патча (то что было выше, оператот Собеля). Для каждого пикселя вычисляется магнитуда (сила) и ориентация (направление) градиента

Далее патч 16×16 делится на 16 подобластей размером 4×4 пикселя. В каждой подобласти строится гистограмма из 8 бинов (направлений). Каждый пиксель влияет своим значением магнитуды в соответствующий бин ориентации. Результат - 8 чисел на каждую подобласть

[<img width="1170" height="519" alt="image" src="https://github.com/user-attachments/assets/cde42b60-999a-43bf-998f-21a8df51a9ad" />]

Формирование итогового вектора. Все 16 гистограмм (по 8 значений) объединяются в один вектор и получается дескриптор размерностью 128 элементов (16 × 8).

Далее делаем нормализацию и ограничение значений. Гистограммы нормализуются для инвариантности к освещению (необходимо для определения всех значений в диапазон 0-1, и их сумма квадратов = 1). Далее значения ограничиваются порогом 0.2 (классический SIFT ограничивает значения, чтобы уменьшить влияние сильных градиентов). Дискриптор становится более устойчивым к изменению освещения.

## Сопоставление дескрипторов между кадрами
Для каждого дескриптора на первом изображении ищется наиболее похожий дескриптор на втором изображении. Похожесть измеряется евклидовым расстоянием ( измеряет кратчайшее расстояние ((d1 и d2) между двумя точками в пространстве) в 128-мерном пространстве. Для дескриптора из первого изображения находятся два ближайших соседа во втором.

Если наилучшее совпадение значительно лучше второго (d1 значительно меньше d2, d1 должен быть меньше 0.75 × d2), то совпадение считается надежным. Порог 0.75 отсеивает неоднозначные сопоставления.

## Построение траектории
RANSAC (RANdom SAmple Consensus) - это итеративный метод оценки параметров математической модели, устойчивый к выбросам. Будем проводить оценку модели не по всеми данными, а только по тем которые не содержат выбросов. Однако поскольку мы незнаем какие данные выбросы а какие нет, то будем брать минимальное количество данных нужных для определения модели, по ним строить модель и дальше искать лучшую. 

Количество итераций: 100 (чем больше, тем выше вероятность найти правильную гипотезу)

Порог ошибки: 10 пикселей (точка считается inlier, если предсказанное положение отличается от реального менее чем на 10 пикселей)
